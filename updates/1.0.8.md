# What has been updated?

## Changelog - (Added, Fixed, Removed):

1. Fixed an issue when renaming the clan, there was no length limit and that would allow to change the clan name to a name with no length limit and abuse it.
2. Added a new abuse protection against clan member kick and hiring - the clan owners can abuse kicking members who are offline to invite online members for their activities to skip buying upgrades.

## Modified Files:

- server/main.lua
- server/functions.lua
- server/nui/management.lua
- locales.lua
- config/config.lua
- fxmanifest.lua

## Notes

1. Add new locales:

```lua
['MAX_LENGTH'] = { text = "The text is bigger than the expected / allowed length.", duration = 4}, -- version 1.0.8
```

2. Add the new database column for the kicked members (Run this on query on the database)

```lua
ALTER TABLE `tp_clans`
ADD COLUMN `kicked_members` LONGTEXT DEFAULT '[]';
```

3. Add the new config option on `config/config.lua` related to kicked members for being invited again.

```lua
-- The clan owners can abuse kicking members who are offline to invite online members for their activities to skip buying upgrades,
-- add the desired cooldown to invite a member again after being kicked out in order to prevent that.
Config.AllowInviteMembersAgainOnceKickedAfter = 48 -- time in hours (48 hours - 2 real-time days by default ) version 1.0.8
```

## How to update?

- You can easily update the modified files by redownloading from the **Cfx.re Portal** and copy-paste on the mentioned directory file paths.
- If new options have been implemented to the configuration file or edited, **DO NOT** copy-paste the new configuration file, otherwise you will get the latest configuration but you will lose all the changes you have made. The proper way updating a configuration file is by finding the new configuration options or the modified ones and do it **manually**.

*Cfx.re Portal **DOES NOT** force-update your configuration file or any files, if it did, you would always lose all the configuration changes you made or would totally break the script, all files are updated or edited manually.*
