# What has been updated?

## Changelog - (Added, Fixed, Removed):

1. Fixed an issue with captured territories cooldown that was not properly removing the cooldown - added an extra option on `config/config_territories.lua` to remove cooldown based on the desired time.
2. Fixed an issue where players could create clans near territories in purpose to control them easily.
3. Fixed reposition abuses on towns, near other territories or clans that the system would not check and would allow this reposition of a clan when should not.
4. Fixed an issue when a player was creating a new clan, he could not do any actions related to territories, furnitures or blackmarket - the player had to relog.

## Modified Files:

- client/main.lua
- server/main.lua
- server/functions.lua
- server/territories.lua
- config/config_territories.lua
- locales.lua
- fxmanifest.lua

## Notes

1. Add on `config/config_territories.lua` the new config for removing captured territories cooldown:

```lua
-- How often should the system remove cooldown from raided territories? The best is between 5-10 minutes.
Config.UpdateTerritoriesCooldownDurationEvery = 5 -- time in minutes ( version 1.1.0 )
```

2. Add on `config/config_territories.lua` the new config for the distance between clans and territory controls.

```lua

-- How much should be the distance from a clan > nearest territory control? 
-- We dont want to allow clans to be repositioned or created near territory controls.
Config.DistanceBetweenClansAndTerritories = 400.0 -- ( version 1.1.0 )
```

3. Add new locales:

```lua
['CANNOT_REPOSITON_OTHER_CLAN_NEARBY']        = {text = "You cannot reposition the clan to this location while another clan(s) are nearby.", duration = 5 }, -- version 1.1.0
['CANNOT_REPOSITON_NEAR_TOWN']                = {text = "You cannot reposition the clan to this location, there is a town nearby.", duration = 5 }, -- version 1.1.0
['CANNOT_REPOSITON_TERRITORY_CONTROL_NEARBY'] = {text = "You cannot reposition the clan to this location while another territory control is nearby.", duration = 5 }, -- version 1.1.0
['CREATE_CLAN_TERRITORY_CONTROL_NEARBY']      = {text = "You cannot create a clan to this location while another territory control is nearby.", duration = 5 }, -- version 1.1.0
    
```

## How to update?

- You can easily update the modified files by redownloading from the **Cfx.re Portal** and copy-paste on the mentioned directory file paths.
- If new options have been implemented to the configuration file or edited, **DO NOT** copy-paste the new configuration file, otherwise you will get the latest configuration but you will lose all the changes you have made. The proper way updating a configuration file is by finding the new configuration options or the modified ones and do it **manually**.

*Cfx.re Portal **DOES NOT** force-update your configuration file or any files, if it did, you would always lose all the configuration changes you made or would totally break the script, all files are updated or edited manually.*
