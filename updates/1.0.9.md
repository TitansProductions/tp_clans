# What has been updated?

## Changelog - (Added, Fixed, Removed):

1. Added blacklisted jobs when a player is attempting to create a clan because the blacklisted jobs were checked only on inviting new members.
2. Modified data triggers from using `TriggerClientEventAsyncByCoords` to `TriggerClientClanUpdateEvent` on not required data to be sent for everyone but only on the updated clan.
3. Fixed an issue when kicking a member from the clan this member still had access to the clan menu and was considered in the clan until this person had to relog than immediately be removed from everywhere.
4. Added a notification to be sent on the kicked member because this person was never notified if was online that he / she got kicked out from a clan.
5. Fixed an issue when adding a new clan member, the new player was not added for all online clan members until they had to relog.
6. Modified `PLAYER_TARGET_INVITE_COOLDOWN` locale to not display minutes but converted time, such as `"1 day and 30 minutes left"`.
7. Fixed multiple issues and errors on territories related to blips, notifications and general updates that would not work properly.
8. Fixed on territories configuration file some minute comment displays for bandit attacks that were wrong.
9. Added an extra feature on clan management to allow or not allow the clan object reposition.
10. Added a new prompt to allow the leader (owner) of a clan to reposition the clan object placement x,y,z,pitch,roll,yaw.

## How to update?

- You can easily update the modified files by redownloading from the **Cfx.re Portal** and copy-paste on the mentioned directory file paths.
- If new options have been implemented to the configuration file or edited, **DO NOT** copy-paste the new configuration file, otherwise you will get the latest configuration but you will lose all the changes you have made. The proper way updating a configuration file is by finding the new configuration options or the modified ones and do it **manually**.

*Cfx.re Portal **DOES NOT** force-update your configuration file or any files, if it did, you would always lose all the configuration changes you made or would totally break the script, all files are updated or edited manually.*
